# Experience Design Principles

> **Anime Calendar v2 is an open-source Anime Release Intelligence Platform dedicated to accuracy, transparency, personalization, and long-term maintainability.**

## Understandable in five seconds

The primary screen should immediately answer what is releasing, what matters to the user, and what changed.

## Every surface answers a question

A screen, widget, or notification must help the user make a decision. Data without a clear purpose does not earn space.

## Calm by default

Avoid clutter, artificial urgency, engagement traps, and unnecessary interruption. Prioritize useful information and visual hierarchy.

## Explain important claims

Recommendations, predictions, changes, and warnings should expose their evidence and confidence when available.

## Customization without fragmentation

Users own widget selection, order, visibility, size, and theme. Customization changes presentation, not the meaning of the underlying data.

## Web first, not web only

The web dashboard is the primary experience. Desktop and API clients reuse the same contracts rather than forking business rules.

## Accessible from the beginning

Keyboard navigation, semantic structure, readable contrast, reduced-motion support, and screen-reader meaning are release requirements, not future polish.

## Respect attention

Notify only when the information is timely, meaningful, and requested by the user.
