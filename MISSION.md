# Mission

> **Anime Calendar v2 is an open-source Anime Release Intelligence Platform dedicated to accuracy, transparency, personalization, and long-term maintainability.**

## Mission statement

Anime Calendar v2 provides anime fans with accurate, trustworthy, and personalized information about what is releasing, when it releases, where it can be watched legally, and which version is available.

The project reduces uncertainty by combining source-aware release data, streaming intelligence, transparent confidence, and user-controlled personalization in one extensible platform.

## Vision

Become the definitive open-source anime release intelligence platform: the trusted first place fans and developers use to understand anime availability across formats, regions, providers, and language versions.

## The five questions

Every major capability should improve at least one of these answers.

### What?

What is releasing: episodes, movies, OVAs, ONAs, specials, shorts, music anime, and future release types.

### When?

When it releases: exact time, exact date, reported date, estimate, delay, cancellation, or unknown.

### Where?

Where it can be watched legally: Crunchyroll, HIDIVE, Netflix, Hulu, Disney+, Prime Video, official sites, theaters, and future providers.

### Which?

Which version is available: original, sub, dub, theatrical, streaming, physical, director's cut, or another clearly modeled variant.

### Why?

Why the system believes the information: official source, AniList record, provider link, curated knowledge, historical model, or another explicit evidence record.

## Product values

1. **Accuracy before speed.** Incomplete but honest data is better than confident misinformation.
2. **Transparency before convenience.** Estimates must be labeled, sourced, and assigned confidence.
3. **User control.** Personalization should reflect user choices, not hidden engagement incentives.
4. **Privacy by design.** Personal data stays separate from public anime metadata and remains exportable and deletable.
5. **Open by default.** Models, architecture, schemas, and public knowledge should be accessible whenever legally and ethically possible.
6. **Reusable foundations.** Calendars, dashboards, APIs, notifications, and future clients consume the same canonical models.
7. **Long-term maintainability.** Each release should improve both capability and foundation.

## North star

> **Every anime fan should be able to answer what releases, when it releases, where to watch it, which version is available, and why the information can be trusted—from one source.**

## Success means

A user can confidently:

- subscribe to a complete or filtered release calendar;
- track episodes and non-episodic anime releases;
- identify legal streaming availability;
- distinguish confirmed, reported, estimated, delayed, cancelled, and unknown dates;
- track sub and dub releases independently;
- understand the source and confidence of important claims;
- create a private profile and control personal filters;
- export or delete personal data;
- use the platform without surrendering attention or privacy.
